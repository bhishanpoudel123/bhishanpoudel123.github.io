Installing postgresql on MacOS Mojave

  • First install pgadmin4 separately, copy the app to Applications.
  • Then, go to postgresql download website
  • Choose 11.5 for macOS which is the latest version.
  • Install the dmg file, UNCHECK pgadmin4, give password and other credentials. Save credentials at ~/.postgres_conf.yaml.
  • I have example postgresql database called dvdrental.tar here in github, download it to ~/Datasets/SQL/dvdrental.tar, do not untar it, just download it to ~/Datasets/SQL/dvdrental/dvdrental.tar. I can double click and open the .dat files inside it in office computer Yosemite, but in my laptop Mojave double click can not open it, also tar -xvz does not work. Basically Mojave have no any method to uncompress that file that it to my knowledge.
  • Open the app pgAdmin, it will show the servers icon on left hand side.
  • For some reasons, pgadmin still shows postgres10, it is crossed by red cross sign, it is unused, just ignore it. postgres11 is blue and shows its active and running good.
  • Servers > PostgreSQL11 > Databases > right click > Create database > give name dvdrental.
  • Servers > Databases > dvdrental > right click > restore > Go to file name and click ... button to go to Finder and locate the file dvdrental.tar, it will be hidden, but change Format backupt to All files in right bottom corner of window and dvdrental.tar is shown, select it.
  • This shows errors, ignore it.
  • Restart pgAdmin couple of times, it shows error, but if you see tables you are fine. (Confirmed worked: Sep 12, 2019)
  • Go to dvdrental > Schemas > Tables > actor > Column to see the columns for the table actor of database dvdrental.
  • Right click dvdrental database > Query tool
  • Do some queries like SELECT * FROM customer limit 5;
  • Now go to your favorite conda env and install psycopg2 and setup sql magic in jupyter notebook.
  • WARNING: In this dvdrental file the table staff is corrupted, it runs fine in pgadmin but fails in jupyter notebook.

This method did not worked for me in July 2021

Ref: https://stackoverflow.com/questions/43641362/adding-syntax-highlighting-to-jupyter-notebook-cell-magic/43686552

Put following text in ~/.jupyter/custom/custom.js

require(['notebook/js/codecell'], function(codecell) {
  codecell.CodeCell.options_default.highlight_modes['magic_text/x-mssql'] = {'reg':[/^%%sql/]} ;
  Jupyter.notebook.events.one('kernel_ready.Kernel', function(){
  Jupyter.notebook.get_cells().map(function(cell){
      if (cell.cell_type == 'code'){ cell.auto_highlight(); } }) ;
  });
});

Using jupyter cell magic %%sql and inline magic %sql

  • These magics are provided by ipython-sql module.
  • postgres requires additional psycopg2 module from conda not pip.
  • sqlite needs additional moduel sqlite3
%load_ext sql
%config SqlMagic.displaycon=False
%config SqlMagic.feedback=False  # no Done, rows affected
%config SqlMagic.autopandas=True

# NOTE: %sql does not need pyspark, it needs only postgres, sqlite, msserver etc

# for sqlite we need to install sqlite
%sql sqlite::/
%sql sqlite:////Volumes/Media/github/Tutorials_and_Lessons/SQL/a05_sqlite3/mydb.db


# for postgres we need to install psycopg2 and pgadmin MUST be running
import os,json
data = json.load(open(os.path.expanduser('~/.config/postgres/credentials.json')))
pw,port = data['password'], data['port']

%sql postgres://postgres:$pw@localhost:$port/try # user=postgres, localhost=localhost, port=5432

Jupyter notebook code format for SQL

  • Open jupyter notebook in dataSc environment where I have installed nbextension configurator.
  • Edit > nbextensions config > code_prettify > json defining library

Paste following code:

{
  "python": {
    "library": "import json\r\n \r\ndef code_reformat(cell_text):\r\n    import yapf.yapflib.yapf_api\r\n    import sqlparse\r\n    import re\r\n \r\n    pyspark_sql_head = \"#%%sql\"\r\n    if cell_text.startswith(pyspark_sql_head):\r\n        sql = cell_text.split(pyspark_sql_head)\r\n        pat = r\"\"\"\\\"\\\"\\\"(.*?)\\\"\\\"\\\"|'''(.*?)'''|\"(.*?)\"|'(.*?)'\"\"\"\r\n        sql = re.search(pat,cell_text,flags=re.MULTILINE|re.DOTALL)[0]\r\n        sql = sql.replace(\"'\",'').replace('\"','')\r\n        formatted_sql = sqlparse.format(sql, keyword_case='upper',reindent=True)\r\n\r\n        pat2 = r\"(.*)\" + re.escape('\"\"\"' + sql + '\"\"\"') + \"(.*)\"\r\n        pre = re.match(pat2,cell_text,flags=re.DOTALL).groups()[0]\r\n        post = re.match(pat2,cell_text,flags=re.DOTALL).groups()[1]\r\n        formatted_post = yapf.yapflib.yapf_api.FormatCode(post)[0]\r\n\r\n        reformatted_text = pre + '\"\"\"' + formatted_sql + '\\n\"\"\"\\n'+ formatted_post\r\n\r\n        return reformatted_text\r\n\r\n    def sqlmatch(match):\r\n        return '%sql'+sqlparse.format(match.group(1), keyword_case='upper')\r\n \r\n    comment = '--%--' if cell_text.startswith('%%sql') else '#%#'\r\n    cell_text = re.sub('^%', comment, cell_text, flags=re.M)\r\n    \r\n    reformatted_text = yapf.yapflib.yapf_api.FormatCode(cell_text)[0] if comment=='#%#' else sqlparse.format(cell_text, keyword_case='upper', reindent=True)\r\n    \r\n    reformatted_text = re.sub('^{}'.format(comment), '%', reformatted_text, flags=re.M)\r\n    if not cell_text.startswith('%%sql'):\r\n        reformatted_text=re.sub('%sql(.*)',sqlmatch , reformatted_text, re.MULTILINE)\r\n    return reformatted_text",
